# Blog Post Quality Checklist

Use this checklist to validate blog post quality at each stage of the generation process.

---

## Pre-Draft Checklist

### Content Extraction
- [ ] All PDF slides processed
- [ ] All topics identified
- [ ] All exercises captured
- [ ] All mathematical formulas extracted
- [ ] All diagrams described for recreation
- [ ] All tables identified
- [ ] Transcript content parsed (if available)
- [ ] Textbook references mapped
- [ ] External resources identified

### Lesson Plan
- [ ] All main topics included
- [ ] Topics ordered logically
- [ ] Prerequisites identified
- [ ] Learning objectives defined
- [ ] Reading time estimated: 15 minutes
- [ ] Structure suitable for blog post

---

## Draft Checklist

### Frontmatter
- [ ] `layout: post` specified
- [ ] Title is descriptive and SEO-friendly (50-70 characters)
- [ ] Date matches lecture date (YYYY-MM-DD format)
- [ ] Categories appropriate (2-3 categories)
- [ ] Tags relevant (5 tags, include course tag)
- [ ] Excerpt compelling (50-100 words)
- [ ] `reading_time: 15` specified
- [ ] `course:` specified correctly

### Introduction
- [ ] Hook: Why is this topic important?
- [ ] Context: How does it fit the course?
- [ ] Preview: What will readers learn?
- [ ] Length: 2-3 paragraphs
- [ ] Engaging and clear

### Main Content
- [ ] All sections from lesson plan included
- [ ] Logical flow between sections
- [ ] Concepts build on each other
- [ ] Technical terms defined before use
- [ ] Examples are clear and illustrative
- [ ] Step-by-step explanations provided
- [ ] Rules/axioms explicitly stated
- [ ] Mathematical notation explained

### Mathematical Content
- [ ] Inline math: `$formula$`
- [ ] Display math: `$$formula$$`
- [ ] Notation explained before first use
- [ ] Formulas are correct
- [ ] Follows @math-rules.md standards
- [ ] Step-by-step derivations shown
- [ ] Intermediate steps included

### Examples
- [ ] At least 2-3 examples per major concept
- [ ] Examples progress from simple to complex
- [ ] Examples include solutions
- [ ] Examples are explained step-by-step
- [ ] Examples illustrate key concepts
- [ ] Both positive and negative examples (if applicable)

### Visual Elements
- [ ] At least 1 diagram per major topic
- [ ] Diagrams are clear and uncluttered
- [ ] Diagrams are properly labeled
- [ ] Diagrams referenced in text
- [ ] Tables formatted correctly
- [ ] Tables explained in text
- [ ] Code examples syntax-highlighted
- [ ] Code examples commented
- [ ] Alt text provided for images

### Exercises
- [ ] All exercises from lecture included
- [ ] Solutions or solution approaches provided
- [ ] Exercises are clear and well-formatted
- [ ] Exercises test key concepts

### External Resources
- [ ] 3-5 high-quality resources included
- [ ] Resources are relevant
- [ ] Resources are authoritative
- [ ] Resources are accessible
- [ ] Resources are properly formatted
- [ ] Brief annotations provided

### Conclusion
- [ ] Summarizes key concepts
- [ ] Connects to broader course themes
- [ ] Suggests next steps or related topics
- [ ] Length: 1-2 paragraphs
- [ ] Provides closure

### Writing Quality
- [ ] Professional but approachable tone
- [ ] Technical precision without jargon overload
- [ ] Active voice preferred
- [ ] One idea per paragraph
- [ ] Short, clear sentences
- [ ] Proper grammar and spelling
- [ ] Consistent terminology

### Structure
- [ ] Single H1 (title)
- [ ] H2 for main sections
- [ ] H3 for subsections
- [ ] Logical heading hierarchy
- [ ] Sections appropriately sized
- [ ] Smooth transitions between sections

### Length and Reading Time
- [ ] Word count: 2000-2500 words
- [ ] Reading time: 15 minutes (±2 minutes)
- [ ] Content density appropriate
- [ ] Not too brief, not too verbose

---

## Post-Review Checklist

### Accuracy
- [ ] All definitions match lecture content
- [ ] Mathematical formulas are correct
- [ ] Examples are accurate
- [ ] Step-by-step solutions are correct
- [ ] No factual errors
- [ ] Terminology consistent with course
- [ ] Cross-referenced with source materials

### Completeness
- [ ] All topics from lesson plan covered
- [ ] All exercises addressed
- [ ] Prerequisites mentioned or explained
- [ ] Key concepts fully explained
- [ ] Examples cover different aspects
- [ ] External resources relevant

### Readability
- [ ] Introduction sets context clearly
- [ ] Logical flow between sections
- [ ] Concepts build appropriately
- [ ] Technical terms defined
- [ ] Examples clear and explained
- [ ] Mathematical notation explained
- [ ] Paragraphs appropriately sized
- [ ] Reading time matches target

### Visual Quality
- [ ] Diagrams accurate and clear
- [ ] Tables well-formatted
- [ ] Code syntax-highlighted
- [ ] Visuals support text
- [ ] Diagrams/tables referenced
- [ ] Alt text provided

### Structure
- [ ] Frontmatter complete and correct
- [ ] Headings follow hierarchy
- [ ] Sections appropriately sized
- [ ] Conclusion synthesizes key points
- [ ] External resources section present

## Final Pre-Publication Checklist

### Format
- [ ] File name: `YYYY-MM-DD-title.md`
- [ ] Ready for `_posts/` directory
- [ ] All assets properly referenced
- [ ] No broken links
- [ ] Images in correct directory

### Content
- [ ] All review feedback addressed
- [ ] Both agents approve
- [ ] Quality standards met
- [ ] Ready for publication

### Assets
- [ ] All diagrams created/saved
- [ ] All tables formatted
- [ ] All screenshots captured (if needed)
- [ ] All assets properly referenced
- [ ] Asset paths correct

### Metadata
- [ ] Metadata.json updated
- [ ] Lecture marked as complete
- [ ] Any special notes documented

## Quality Metrics

### Quantitative Targets
- **Word Count:** 2000-2500 words
- **Reading Time:** 15 minutes (±2 minutes)
- **Diagrams:** At least 1 per major topic
- **Examples:** At least 2-3 per major concept
- **External Resources:** 3-5 high-quality links
- **Tags:** 5 relevant tags
- **Categories:** 2-3 categories

### Qualitative Standards
- **Accuracy:** All information verified
- **Completeness:** All topics covered
- **Readability:** Clear, logical flow
- **Visual Quality:** Diagrams/tables enhance understanding
- **Engagement:** Informative and valuable

---

## Usage Instructions

1. **During Drafting:** Use "Draft Checklist" to ensure all elements included
2. **After Review:** Use "Post-Review Checklist" to verify feedback addressed
3. **Before Publication:** Use "Final Pre-Publication Checklist" for final validation
4. **Course-Specific:** Check relevant course-specific items

**Note:** Not all items may apply to every post. Use judgment to determine relevance.
